Bucketing and Mapping: Categories and Bottles

TLDR: This step builds a couple of useful derivations, category_class and bottle_size. There as 68 different category_names in this data set, we reduce that to 9. There are 34 liter sizes, we make a new dimension, bottle_size that only has 3 possible values.

Our previous analysis of price per mL brings to mind questions around bottle size. How many different sizes do bottles come in? Are there standards and uncommon ones? Do vendors specialize in different bottle sizes?

Building category_class, a simplified version of category_name

Using the query below, we can see that there are 68 different category names in the data set. We can notice there is 80 Proof Vodka, Flavored Vodka and more. It would be helpful if we just could have all of these categorized together as vodkas.

query: iowa -> {
  aggregate: distinct_number_of_category_names is count(distinct category_name)
  nest: sizes is {
    group_by: category_name
    aggregate: item_count, line_item_count
  }
}

WITH __stage0 AS (
  SELECT
    group_set,
    CASE WHEN group_set=0 THEN
      count(distinct iowa.category_name)
      END as distinct_number_of_category_names__0,
    CASE WHEN group_set=1 THEN
      iowa.category_name
      END as category_name__1,
    CASE WHEN group_set=1 THEN
      count(distinct iowa.item_number)
      END as item_count__1,
    CASE WHEN group_set=1 THEN
      COUNT( 1)
      END as line_item_count__1
  FROM `malloy-data.iowa_liquor_sales.sales_deduped` as iowa
  CROSS JOIN (SELECT row_number() OVER() -1  group_set FROM UNNEST(GENERATE_ARRAY(0,1,1)))
  GROUP BY 1,3
)
SELECT
  ANY_VALUE(CASE WHEN group_set=0 THEN distinct_number_of_category_names__0 END) as distinct_number_of_category_names,
  ARRAY_AGG(CASE WHEN group_set=1 THEN STRUCT(
    category_name__1 as category_name, 
    item_count__1 as item_count, 
    line_item_count__1 as line_item_count
    ) END IGNORE NULLS  ORDER BY  item_count__1 desc ) as sizes
FROM __stage0
ORDER BY 1 desc

Malloy provides a simple way to map all these values, using pick expressions. In the Malloy Model for this Data Set, you will find the declaration below. Each pick expression tests category_name for a regular expression. If it matches, it returns the name pick'ed.

category_class is category_name ?
  pick 'WHISKIES' when ~ r'(WHISK|SCOTCH|BURBON|RYE)'
  pick 'VODKAS' when ~ r'VODKA'
  pick 'RUMS' when ~ r'RUM'
  pick 'TEQUILAS' when ~ r'TEQUILA'
  pick 'LIQUEURS' when ~ r'(LIQUE|AMARETTO|TRIPLE SEC)'
  pick 'BRANDIES' when ~ r'BRAND(I|Y)'
  pick 'GINS' when ~ r'GIN'
  pick 'SCHNAPPS' when ~ r'SCHNAP'
  else 'OTHER'

  SELECT
    group_set,
    CASE WHEN group_set IN (0,1) THEN
      CASE WHEN REGEXP_CONTAINS(iowa.category_name, r'(WHISK|SCOTCH|BOURBON|RYE)') THEN 'WHISKIES' WHEN REGEXP_CONTAINS(iowa.category_name, r'VODKA') THEN 'VODKAS' WHEN REGEXP_CONTAINS(iowa.category_name, r'RUM') THEN 'RUMS' WHEN REGEXP_CONTAINS(iowa.category_name, r'TEQUILA') THEN 'TEQUILAS' WHEN REGEXP_CONTAINS(iowa.category_name, r'(LIQUE|AMARETTO|TRIPLE SEC)') THEN 'LIQUEURS' WHEN REGEXP_CONTAINS(iowa.category_name, r'BRAND(I|Y)') THEN 'BRANDIES' WHEN REGEXP_CONTAINS(iowa.category_name, r'GIN') THEN 'GINS' WHEN REGEXP_CONTAINS(iowa.category_name, r'SCHNAP') THEN 'SCHNAPPS' ELSE 'OTHER' END
      END as category_class__0,
    CASE WHEN group_set=0 THEN
      count(distinct iowa.item_number)
      END as item_count__0,
    CASE WHEN group_set=1 THEN
      iowa.category_name
      END as category_name__1,
    CASE WHEN group_set=1 THEN
      count(distinct iowa.item_number)
      END as item_count__1
  FROM `malloy-data.iowa_liquor_sales.sales_deduped` as iowa
  CROSS JOIN (SELECT row_number() OVER() -1  group_set FROM UNNEST(GENERATE_ARRAY(0,1,1)))
  GROUP BY 1,2,4
)
SELECT
  category_class__0 as category_class,
  ANY_VALUE(CASE WHEN group_set=0 THEN item_count__0 END) as item_count,
  ARRAY_AGG(CASE WHEN group_set=1 THEN STRUCT(
    category_name__1 as category_name, 
    item_count__1 as item_count
    ) END IGNORE NULLS  ORDER BY  item_count__1 desc ) as names_list
FROM __stage0
GROUP BY 1
ORDER BY 2 desc

WITH __stage0 AS (
  SELECT
    group_set,
    __lateral_join_bag.category_class__0,
    CASE WHEN group_set=0 THEN
      COALESCE(SUM(iowa.sale_dollars),0)
      END as total_sale_dollars__0,
    __lateral_join_bag.vendor_name__1,
    __lateral_join_bag.item_description__1,
    CASE WHEN group_set=1 THEN
      COALESCE(SUM(iowa.sale_dollars),0)
      END as total_sale_dollars__1,
    CASE WHEN group_set=1 THEN
      COALESCE(SUM(iowa.bottles_sold),0)
      END as total_bottles__1,
    CASE WHEN group_set=1 THEN
      AVG((iowa.state_bottle_retail/nullif(iowa.bottle_volume_ml,0)*100))
      END as avg_price_per_100ml__1,
    (CASE WHEN group_set=1 THEN
      COALESCE(SUM(iowa.sale_dollars),0)
      END)/MAX((CASE WHEN group_set=0 THEN
      COALESCE(SUM(iowa.sale_dollars),0)
      END)) OVER (PARTITION BY __lateral_join_bag.category_class__0)*100 as percent_of_sales__1
  FROM `malloy-data.iowa_liquor_sales.sales_deduped` as iowa
  CROSS JOIN (SELECT row_number() OVER() -1  group_set FROM UNNEST(GENERATE_ARRAY(0,1,1)))
  LEFT JOIN UNNEST([STRUCT(CASE WHEN group_set IN (0,1) THEN
    CASE WHEN REGEXP_CONTAINS(iowa.category_name, r'(WHISK|SCOTCH|BOURBON|RYE)') THEN 'WHISKIES' WHEN REGEXP_CONTAINS(iowa.category_name, r'VODKA') THEN 'VODKAS' WHEN REGEXP_CONTAINS(iowa.category_name, r'RUM') THEN 'RUMS' WHEN REGEXP_CONTAINS(iowa.category_name, r'TEQUILA') THEN 'TEQUILAS' WHEN REGEXP_CONTAINS(iowa.category_name, r'(LIQUE|AMARETTO|TRIPLE SEC)') THEN 'LIQUEURS' WHEN REGEXP_CONTAINS(iowa.category_name, r'BRAND(I|Y)') THEN 'BRANDIES' WHEN REGEXP_CONTAINS(iowa.category_name, r'GIN') THEN 'GINS' WHEN REGEXP_CONTAINS(iowa.category_name, r'SCHNAP') THEN 'SCHNAPPS' ELSE 'OTHER' END
    END as category_class__0,
  CASE WHEN group_set=1 THEN
    iowa.vendor_name
    END as vendor_name__1,
  CASE WHEN group_set=1 THEN
    iowa.item_description
    END as item_description__1)]) as __lateral_join_bag
  GROUP BY 1,2,4,5
)
SELECT
  category_class__0 as category_class,
  ANY_VALUE(CASE WHEN group_set=0 THEN total_sale_dollars__0 END) as total_sale_dollars,
  ARRAY_AGG(CASE WHEN group_set=1 THEN STRUCT(
    vendor_name__1 as vendor_name, 
    item_description__1 as item_description, 
    total_sale_dollars__1 as total_sale_dollars, 
    total_bottles__1 as total_bottles, 
    avg_price_per_100ml__1 as avg_price_per_100ml, 
    percent_of_sales__1 as percent_of_sales
    ) END IGNORE NULLS  ORDER BY  total_sale_dollars__1 desc  LIMIT 5) as top_sellers_by_revenue
FROM __stage0
GROUP BY 1
ORDER BY 2 desc

Creating a new Dimension for Bottle Size.

Looking at the above chart and table we can see that there are a bunch of small values, several big values at 750 and 1000, and then a bunch of larger values. We can clean this up by bucketing bottle size into three groups using a Malloy pick expression that maps these values to strings.

bottle_size is bottle_volume_ml ?
  pick 'jumbo (over 1000ml)' when > 1001
  pick 'liter-ish' when >= 750
  else 'small or mini (under 750ml)'

Look at the data through the new mapping.

  aggregate: total_sale_dollars, line_item_count, item_count
  order_by: bottle_size
